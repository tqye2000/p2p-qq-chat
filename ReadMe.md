# 安全的点对点加密聊天应用 (P2P 悄悄话)

一个简单的点对点加密聊天应用程序，使用 WebRTC（通过 PeerJS）和 Web Crypto API。无需服务器存储消息 - 消息直接从一个浏览器传输到另一个浏览器，完全加密。

## 🔒 功能特点

- **无服务器数据存储**：聊天消息通过 WebRTC 点对点连接直接从用户 A 传输到用户 B。没有数据库存储您的聊天记录。
- **端到端加密**：消息在离开您的浏览器之前使用 AES-GCM（256位）加密，只有在到达对方时才解密。
- **纯客户端运行**：所有功能都在您的浏览器中运行。即使有人拦截了点对点传输，他们也只能看到加密数据。
- **无需安装**：完全在浏览器中运行 - 只需打开 HTML 文件即可。
- **持久化 ID 选项**：可选择记住您的对等 ID 以便跨会话使用，或每次生成新的随机 ID。
- **简洁界面**：使用 Tailwind CSS 打造的清爽界面。

## 🚀 使用方法

1. **打开文件**：在两个不同的浏览器标签页中打开 `p2p_qq_chat.html`（或将文件发送给朋友）

2. **连接对等端**：
   - **标签页 1**：复制 "我的 ID"
   - **标签页 2**：将该 ID 粘贴到 "朋友的 ID" 并点击连接

3. **选择 ID 持久化**（可选）：
   - 默认情况下，"记住我的 ID" 复选框已勾选，这意味着您的对等 ID 将被保存并在会话间重复使用
   - 如果您希望每次重新加载页面时使用新的随机 ID，请取消勾选
  
4. **设置加密密钥**：双方用户必须输入**相同**的 "秘密加密密码"
   
5. **开始聊天**：消息现在已在对等端之间进行端到端加密！

## 🔐 安全说明

- 🔒 消息在传输前已加密
- 🔒 只有使用正确密码的用户才能解密消息
- ⚠️ 密码本身永远不会被传输
- ⚠️ 出于演示目的，使用了静态盐值。在生产环境中，应使用安全共享的随机盐值。
- ⚠️ 这是一个演示项目 - 用于实际应用时，请考虑采取额外的安全措施
